Database Management System Using SQL

In my ISDS 402 course, Database Management Systems, I had one individual group project. This project required the usage of SQL and formulating designs such as entities, tables, and the primary and foreign keys. ... This project required me to create a database, test it, and maintain it using code, computer programs, and applications. The assignment included eight steps.

i. Step 1.) I had to create as many entities as I needed to match the professor’s requirements and include the attributes that were asked in the project’s description. I created 6 entities with several attributes. Each entity had a primary key because I needed them to create foreign keys for certain entities that eventually connected all of them. I also created assumptions which are restraints that the entities require in order to work or function correctly.

ii. Step 2.) I created all the entities’ diagrams and used Crow Foot’s notation to connect each entity. I also provided a description of the diagrams, what crow foot notation was included, and why it was. There were 12 different relationships and I used MS Visio to create these diagrams.

iii. Step 3.) I created a matrix that included the fields, data type, field size, and description of each attribute for each entity. I created 6 matrixes.

iv. Step 4.) I created a script that had “create data tables and insert values.” This was to provide the foundation of the database and what it included. I had to type over 200 lines of script of unique data for each attribute of each entity. Then I inputted them into SDL Management Studio where it generated a database and tables that I can review or look over when searching the database for certain data using SQL commands.

v. Step 5.) This was to look over the SQL Management Studio’s Diagrams and make sure that they were correct and accurate. I needed to look at the tables and diagrams to ensure that there were no redundancies or mistakes.

vi. Step 6.) I had to create an SQL statement that selected all the entities and displayed it at the bottom of the UI to ensure that the SQL commands could locate my database.

vii. Step 7.) I created several SQL statements such as “List the total sales for each dealer over a month’s time frame or List the amount due from NHD to each dealer after all deductions are considered in a particular month.” I provided excellent proficiency in the SQL codes as it pulled accurate and correct data.

viii. Step 8.) Answer all the professor’s questions in detail. The questions were about normalization and why it was a significant factor when defining a database design, how I validated a proposed database design model before implantation, what challenges I faced during the project, and what advice I could give to future ISDS 402 students.
